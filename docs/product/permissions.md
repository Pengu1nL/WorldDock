# 权限模型

WorldDock Cloud Alpha 使用个人账户权限模型。每个登录用户以个人创作者身份管理自己的世界、发布内容、Fork、用量、通知、反馈和个人访问令牌，不提供组织、团队或管理员层级。

## Alpha 用户可管理

- 自己的世界：创建、编辑、删除或导出自己拥有的世界与草稿资产。
- 发布：将自己世界中的指定内容发布为公开 release snapshot，并管理发布说明。
- Fork：从公开 release snapshot 创建自己的副本，后续编辑归 Fork 创建者所有。
- 用量：查看与自己账户相关的额度、AI 调用或导出等 Alpha 用量提示。
- 通知：查看与自己发布、Fork、反馈或系统提示相关的个人通知。
- 反馈：提交与产品体验、发布内容或 AI 协作结果相关的个人反馈。
- PAT：创建、查看、撤销自己的 Personal Access Token，用于个人自动化或 API 调用。

## Beta 暂缓

- 组织和团队成员。
- 项目级角色。
- 管理员角色。
- 审核员工作台。
- 付费账户角色。
- 细粒度共享链接权限。

## 验收口径

Alpha 文案、界面、API 命名和产品说明不得暗示团队、管理员、审核员、组织结算或付费角色已经可用。若需要展示未来方向，应明确标注为 Beta 暂缓或后续评估。
